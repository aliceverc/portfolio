Virtual Pet Care (Pocket Pets)
Virtual Pet Care is a mobile-first web app where users adopt colourful digital pets, watch their needs change in real time, and interact with them through feeding, sleeping and playing. Each pet has its own character (balanced, gourmet, relaxed, playful) that sets default hunger / energy / entertainment levels and influences how fast those needs decay.

Design Concept:
  • Phone-Sized Focus: The entire interface lives in a centred 375 px-wide column on larger screens, mimicking a native-app feel and ensuring comfortable thumb reach on mobile.
  • Card-Based Pet Gallery: Each pet appears as a tidy card with a live emoji happiness indicator, name, and a dynamic avatar built from colour, shape and border attributes‚Äîinstantly recognisable at a glance.
  • Playful Colour Accents: Soft pastels for backgrounds and bright accent colours for hunger, energy and entertainment bars reinforce a fun, game-like mood without overwhelming the user.
  • Need Bars & Emoji Feedback: Animated fill bars and a three-state emoji (happy üòÑ, neutral üòê, sad üò¢) give immediate visual feedback on a pet‚Äôs well-being.
  • Minimal Top Nav: A simple logo-button in the top-left corner returns users to the pet list from any page, maintaining a clean, clutter-free header.

Key Features:
  • Create, Edit & Release Pets:
    • Add a new pet with name (max 15 chars), character tag and visual tweaks.
    • Edit existing pets; character changes instantly re-seed default needs.
    • Release (delete) pets with an ‚ÄúAre you sure?‚Äù dialog.
  • Character-Based Defaults: Four personality tags prefill needs and affect decay ‚Äî e.g. playful pets lose entertainment faster than relaxed pets.
  • Real-Time Interaction Buttons: ‚ÄúEat,‚Äù ‚ÄúSleep,‚Äù ‚ÄúPlay‚Äù adjust needs immediately with a quick bar-flash animation; buttons disable while an update is in flight.
  • Automatic Need Decay: A background timer (or revalidation on page load) decreases needs; full depletion triggers silent removal from the DB.
  • Responsive, Touch-Friendly Layout: Thumb-sized buttons, single-column flow, and generous spacing ensure comfortable use on any phone.
  • Empty-State Messaging: When all pets are gone, the list view shows ‚ÄúYou have no pets yet‚Äù plus a friendly ‚ÄúAdd a new Pet‚Äù call-to-action.

Technical Execution:
  • Next.js 14 + React 18: CSR pages with /api routes for CRUD operations.
  • MongoDB Atlas + Mongoose: Schema stores appearance, character and needs; findByIdAndUpdate & findByIdAndDelete power edit / release workflows.
  • SWR Data Fetching & Cache: Automatic revalidation after updates; mutate() ensures UI reflects the latest DB state.
  • styled-components: Encapsulated styling, easy theming and conditional flash animations on need bars.
  • Mobile-First Layout: Global container max-width: 375px; margin: 0 auto; keeps the UI consistent across viewports.
  • Fast Feedback Loops: fetch + optimistic UI updates; no page reloads.
  • Clean Code & Modularity: Dedicated components for PetCard, PetDisplay, NeedsBar, PetHappiness, Logo, etc., each with clear props and responsibilities.


Skills: React / Next.js routing & API routes, MongoDB schema design & Mongoose queries (CRUD), SWR for real-time data and optimistic UI, styled-components for responsive and mobile-first design, Form validation (max-length, required character tag), UX patterns (confirmation dialogs, empty-state messaging, centred fixed-width layout), Git feature-branch workflow & Pull Requests
